PRD: Solana Program & Token-2022 Authority Audit Tool

1. Overview

A TypeScript CLI tool that produces a structured authority audit report for any pair of (Solana program, Token-2022 mint). Given a program ID and a mint address, it resolves every authority slot involved in governing the program and the token, classifies each authority holder (EOA / Squads multisig / SPL Governance PDA / None), and outputs a JSON report suitable for human review and version control.

The tool's primary use case is auditing transfer-hook-enabled stablecoins and other Token-2022 deployments, where the effective governance surface spans both the mint's extension authorities and the upgrade authority of the hook program.

2. Goals & non-goals

Goals

  • Resolve every authority slot on a program and a Token-2022 mint to a concrete pubkey (or None).
  • Classify each authority pubkey as: EOA, Squads multisig, SPL Governance PDA, other program-owned account, or non-existent.
  • Report whether the program has a verified build (OtterSec) and an Anchor IDL.
  • Produce upgrade history for the program (slot, signer, signature per upgrade).
  • Output a deterministic JSON report that can be diffed across runs to detect authority changes.

Non-goals

  • No source-code analysis of the program. Verification status only.
  • No risk scoring or recommendations. The tool surfaces facts; humans interpret them.
  • No support for non-Token-2022 mints (classic SPL Token mints are out of scope for v1).
  • No real-time monitoring or webhooks. Single-shot CLI only.

3. Inputs & outputs

Inputs

  • --program <PUBKEY> — required. Program ID to audit.
  • --mint <PUBKEY> — required. Token-2022 mint to audit.
  • --rpc <URL> — optional. RPC endpoint. Defaults to RPC_URL env var.
  • --out <PATH> — optional. Output file path. Defaults to ./audit-<program>-<mint>.json.

Output

Where AuthorityHolder is:

{
  "pubkey": "..." | null,        // null means authority is None / renounced
  "exists": true,                 // is there a system account at this pubkey?
  "onCurve": true,                // ed25519-curve check; off-curve implies PDA
  "classification": "eoa" | "squads_multisig" | "spl_governance" | "program_owned_other" | "none" | "unfunded_eoa",
  "owner": "...",                 // owner program of the account, if exists
  "lamports": 0,
  "details": { /* classification-specific extras, e.g. squads threshold */ }
}

Environment

Create a .env file (and add it to .gitignore):

RPC_URL=https://mainnet.helius-rpc.com/?api-key=YOUR_KEY

The tool should read RPC_URL from the environment if --rpc is not passed. Any standard Solana RPC works (Helius, QuickNode, Alchemy, Triton One). The public api.mainnet-beta.solana.com endpoint may be used but is heavily rate-limited and will likely fail on the upgrade-history step.

6. Implementation plan

The implementation is split into independent modules, each responsible for one section of the report. The CLI orchestrates them and merges the output.

Directory structure

src/
  cli.ts                  // entry point, arg parsing, orchestration
  types.ts                // shared types (AuthorityHolder, AuditReport)
  rpc.ts                  // Connection factory
  program/
    info.ts               // loader, ProgramData, upgrade authority, bytecode hash
    history.ts            // upgrade transaction history
    verified-build.ts     // OtterSec verify API
    anchor-idl.ts         // Anchor IDL fetch
  mint/
    info.ts               // mint + all Token-2022 extensions
  authority/
    classify.ts           // pubkey -> AuthorityHolder
    squads.ts             // Squads multisig detection
    governance.ts         // SPL Governance detection

Module specs

src/rpc.ts

Exports createConnection(rpcUrl?: string): Connection. Resolves URL from arg or RPC_URL env. Throws if neither is set.

src/program/info.ts

Function getProgramInfo(conn, programId):

  1. Call conn.getAccountInfo(programId). Confirm executable === true. Identify loader from owner:
    • BPFLoader1111111111111111111111111111111111 → v1 (immutable, no upgrade authority)
    • BPFLoader2111111111111111111111111111111111 → v2 (immutable)
    • BPFLoaderUpgradeab1e11111111111111111111111 → v3
    • LoaderV411111111111111111111111111111111111 → v4
  2. For v3: derive ProgramData PDA from [programId.toBuffer()] under the loader. Fetch its account.
  3. Parse ProgramData bytes:
    • Bytes 0..4: discriminator (must be 3)
    • Bytes 4..12: slot (u64 LE) → lastDeployedSlot
    • Byte 12: Option discriminator for upgrade authority (0 = None, 1 = Some)
    • Bytes 13..45 (if Some): upgrade authority pubkey
    • Bytes 45..: ELF bytecode
  4. Compute sha256 over the bytecode region. Use Node's crypto.createHash.
  5. For v4: similar logic but different account layout — defer to @solana/web3.js helpers or treat as out of scope for v1 (return a stub and log a warning).
  6. For v1/v2: upgradeAuthority.pubkey = null, mark loader as immutable.

src/program/history.ts

Function getUpgradeHistory(conn, programDataAddress, limit = 100):

  1. conn.getSignaturesForAddress(programDataAddress, { limit }).
  2. For each signature, fetch the transaction with conn.getParsedTransaction(sig, { maxSupportedTransactionVersion: 0 }).
  3. Filter instructions where programId is the BPF Loader Upgradeable and the instruction is an Upgrade or DeployWithMaxDataLen.
  4. Extract slot, fee payer (typically the upgrade authority signer), and tx signature.
  5. Return chronologically ascending.

Note: rate limits will be hit on public RPC. Paginate via the before parameter if needed. If history is unavailable, return an empty array and set a warning field in the report.

src/program/verified-build.ts

Function getVerifiedBuild(programId):

  1. fetch("https://verify.osec.io/status/" + programId).
  2. Parse with zod against expected shape. Handle 404 (not verified) and 5xx (API down).
  3. Return { verified: boolean, repoUrl?, commit?, verifiedAt? }.

src/program/anchor-idl.ts

Function getAnchorIdl(conn, programId):

  1. Anchor IDL PDA: findProgramAddressSync([], programId) gives a base; the actual IDL account is derived as createWithSeed(base, "anchor:idl", programId).
  2. Easier path: use anchor.Program.fetchIdl(programId, provider) with a minimal provider built from conn.
  3. If the IDL account doesn't exist, return { present: false }.
  4. If present, extract name, version (from metadata if available), and instruction names.

src/mint/info.ts

Function getMintInfo(conn, mintAddress):

  1. getMint(conn, mintAddress, "confirmed", TOKEN_2022_PROGRAM_ID) from @solana/spl-token.
  2. Reject if the mint's owner is not the Token-2022 program.
  3. For each known extension type, call the matching getter from @solana/spl-token:
    • getTransferFeeConfig
    • getMintCloseAuthority
    • getPermanentDelegate
    • getInterestBearingMintConfigState
    • getDefaultAccountState
    • getTransferHook
    • getMetadataPointerState
    • getGroupPointerState
    • getGroupMemberPointerState
    • getTokenMetadata
    • getPausableConfig (if available in installed SDK version)
    • getScaledUiAmountConfig (if available)
    • getConfidentialTransferMint
  4. Each getter returns null if the extension is absent. Skip nulls.
  5. Build the extensions array with each non-null extension's type and its authority/delegate field.

src/authority/classify.ts

Function classifyAuthority(conn, pubkey | null): Promise<AuthorityHolder>:

  1. If pubkey is null → return { pubkey: null, classification: "none" }.
  2. Compute onCurve = PublicKey.isOnCurve(pubkey).
  3. conn.getAccountInfo(pubkey).
  4. If account does not exist:
    • onCurve === true → classification: "unfunded_eoa"
    • onCurve === false → still report as non-existent; this is anomalous for a PDA
  5. If account exists, branch on owner:
    • 11111111111111111111111111111111 (System Program) → "eoa"
    • Squads program ID (SQDS..., see @sqds/multisig constants) → "squads_multisig". Fetch the multisig account and populate details with threshold and member count.
    • SPL Governance program ID (GovER5Lthms3bLBqWub97yVrMmEogzX7xNjdXpPPCVZw) → "spl_governance". Populate details with realm.
    • Anything else → "program_owned_other" with owner recorded for follow-up.
  6. Always populate exists, onCurve, owner, lamports.

src/cli.ts

Uses commander to parse args. Calls each module in parallel where possible (program info, mint info, verified build, IDL can all run concurrently). After all complete, walks the result tree, collects unique authority pubkeys, classifies each once (memoized), substitutes AuthorityHolder objects in place, writes JSON output, and prints a summary to stdout.

7. Example invocation

Audit the USDK transfer-hook stablecoin on Solana mainnet:

# Program (transfer hook) and mint addresses
export PROGRAM_ID=extaykYu5AQcDm3qZAbiDN3yp6skqn6Nssj7veUUGZw
export MINT_ID=usdkbee86pkLyRmxfFCdkyySpxRb5ndCxVsK2BkRXwX
export RPC_URL=https://mainnet.helius-rpc.com/?api-key=YOUR_KEY

# Run via tsx during development
pnpm dev --program $PROGRAM_ID --mint $MINT_ID

# Or after build
pnpm build
pnpm start --program $PROGRAM_ID --mint $MINT_ID --out usdk-audit.json

Expected behavior:

  1. Tool fetches the program account, identifies the loader (expected: BPF Loader v3 based on deployment patterns for hook programs).
  2. Derives ProgramData, extracts upgrade authority pubkey, computes bytecode hash.
  3. Fetches the mint, identifies it as Token-2022, enumerates extensions. Expected to find at minimum a TransferHook extension with programId === extaykYu5AQcDm3qZAbiDN3yp6skqn6Nssj7veUUGZw.
  4. Classifies the upgrade authority of the hook program, the mint authority, freeze authority, transfer hook authority, and any other extension authorities present.
  5. Queries OtterSec for verified build status of the hook program.
  6. Attempts to fetch an Anchor IDL.
  7. Writes the report to usdk-audit.json.

Reviewer's next step is to open the JSON, focus on every AuthorityHolder with classification: "eoa" or "unfunded_eoa", and treat those as findings to investigate.

8. Acceptance criteria

  • Running pnpm dev --program <P> --mint <M> against the example USDK pair produces a JSON file with non-empty program and mint objects within 30 seconds on a Helius free-tier endpoint.
  • Every authority field in the output is either null (with classification: "none") or a fully populated AuthorityHolder.
  • No AuthorityHolder has classification set to a value outside the enum.
  • pnpm typecheck passes with zero errors under strict: true.
  • pnpm test passes. Tests must cover: ProgramData byte parsing, Option discriminator handling, classification branching by owner, missing-account handling, and OtterSec 404 handling.
  • Tool exits non-zero with a clear error if the mint is not owned by Token-2022, or if the program account is not executable.
  • Re-running the tool against the same inputs at the same slot produces byte-identical JSON output (modulo auditedAt timestamp).

9. Out-of-scope / future work

  • Classic SPL Token mint support.
  • Loader v4 full support (currently stubbed).
  • Recursive authority resolution (e.g., if a Squads multisig is itself controlled by a DAO, resolve that layer too).
  • Diff mode: compare two reports and highlight changed authorities.
  • Webhook integration for ongoing monitoring.
  • HTML report rendering.
